ELSA: Exact Linear-Scan Attention for Fast and Memory-Light Vision Transformers

Chih-Chung Hsu, Xin-Di Ma, Wo-Ting Liao, Chia-Ming Lee
National Yang Ming Chiao Tung University

CVPR Findings 2026

Paper arXiv Code

🚀 TL;DR

Standard attention is slow and memory-heavy. **ELSA** fixes this by turning attention into a "Parallel Scan" problem. It achieves **O(log n) parallel depth** and **O(n) memory** without losing any precision and requiring **zero retraining**. Drop it into CLIP or SAM and get up to **3.5x speedup** on FP32.

Abstract

Existing attention accelerators often trade exact softmax semantics for speed or suffer from sequential bottlenecks. We present ELSA, an algorithmic reformulation of softmax attention that preserves exact semantics in real arithmetic. By casting the online softmax update into a prefix scan over an associative monoid of triples $(m, S, W)$, ELSA achieves $O(n)$ extra memory and $O(\log n)$ parallel depth. Implemented in Triton and CUDA C++, it improves FP32 inference latency over memory-efficient SDPA by 1.3–3.5x as a drop-in replacement.

Methodology: The Monoid formulation

Standard online attention is sequential. ELSA breaks this by proving that attention states form an Associative Monoid. The state triple consists of:

  • $m$: The running maximum logit.
  • $S$: The accumulated normalization sum.
  • $W$: The weighted sum of value vectors.

Using a two-level scan (intra-block Hillis-Steele and inter-block Blelloch), we reduce the number of global synchronization steps from millions to just ~19 for a 400k token sequence.

Zero-Shot Deployment Results

ELSA is a "Plug-and-Play" solution for foundation models. We observed significant gains in real-world vision tasks:

  • CLIP (ViT-L/14): Halved inference time at 560px resolution with **39.6% VRAM savings**.
  • 3D Vision (VGGT): Achieved **2.34x speedup** over xFormers for long sequence frame aggregation.
  • Embedded (Jetson TX2): Achieved **1.5x-2.0x lower latency** than PyTorch's default Math kernels.
