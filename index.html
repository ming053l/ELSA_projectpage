Abstract

Existing attention accelerators often trade exact softmax semantics or suffer from sequential depth that limits throughput on long sequences. We present ELSA, an algorithmic reformulation of softmax attention that preserves exact semantics in real arithmetic. By casting the online softmax update into a prefix scan over an associative monoid of triples $(m, S, W)$, ELSA achieves $O(n)$ extra memory and $O(\log n)$ parallel depth. Implemented in Triton and CUDA C++ without Tensor Core dependencies, it improves FP32 inference latency over memory-efficient SDPA by 1.3–3.5× as a drop-in replacement, with no retraining required.

Motivation

The key insight behind ELSA comes from an analogy with classical image processing. A Summed-Area Table (SAT)—also known as an integral image—lets you compute the sum of any rectangular region in constant time, regardless of its size, by a single-pass precomputation. Softmax attention faces an analogous bottleneck. The standard online recurrence of Rabe & Staats compresses the $O(n^2)$ attention matrix to $O(n)$ memory, but at the cost of strict sequentiality: each token update depends on all previous ones, forcing $O(n)$ synchronization steps. ELSA resolves this by showing that the attention state $(m, S, W)$ satisfies the same composability property as prefix sums—enabling a fully parallel reduction with $O(\log n)$ depth.

Method: Parallel Monoid Scan

ELSA transforms the sequential online softmax recurrence into a fully parallelizable computation by proving that attention states $(m, S, W)$ form an associative monoid under a merge operator $\oplus$. This algebraic structure is precisely what enables standard parallel-scan primitives—analogous to how prefix sums unlock $O(1)$ range queries in integral images—without any approximation or hardware-specific instructions.

From Sequential Recurrence to Parallel Scan

Classical memory-efficient attention (ME-SDPA) chains tokens one by one: each step depends on the previous running maximum $m_j$, forcing $O(n)$ sequential depth. On a GPU with one million tokens, this means one million synchronization barriers—leaving the vast majority of threads idle. ELSA breaks this chain by observing that any two contiguous blocks of attention states can be merged exactly via:

$u_A \oplus u_B = \mathrm{renorm}\!\left(\max(m_A, m_B),\; \tilde{S}_A + \tilde{S}_B,\; \tilde{W}_A + \tilde{W}_B\right)$

Because $\oplus$ is associative (with identity $e = (-\infty, 0, \mathbf{0})$), blocks can be reduced independently in parallel and combined in a logarithmic reduction tree—exactly as prefix sums compose corner values in a SAT.

Two-Level GPU Scan

We map this monoid to a two-level scan architecture tailored to the GPU memory hierarchy:

  • Intra-block (Hillis–Steele): Each thread block independently reduces 128 tokens in shared memory using a tree scan. All threads remain active throughout, maximizing shared-memory bandwidth.
  • Inter-block (Blelloch): A work-optimal two-pass scan aggregates block-level states in global memory, adding only $2\lceil\log_2(n/B)\rceil$ merge levels. The identity element $e$ allows the down-sweep to initialize boundary states without special-casing.

The total scan depth is $L(n, B) = \lceil\log_2 B\rceil + 2\lceil\log_2(n/B)\rceil + 3 \leq 2\lceil\log_2 n\rceil + 3$, yielding a provable FP32 relative error bound of $O(u \log n)$—logarithmic growth versus the $O(nu)$ of sequential recurrence.

Key Design Properties

  • Algorithmic Parallelization: The attention state is treated as a one-dimensional analogue of an integral image, enabling partial sums to be merged in parallel rather than accumulated sequentially.
  • Hardware Agnosticism: $\oplus$ requires only scalar max/exp and vector fma—no HMMA/GMMA Tensor Core instructions. ELSA runs identically on A100s, older AMD devices, and embedded systems such as the Jetson TX2.
  • Exact Precision: Unlike linear-attention variants that approximate the softmax operator, ELSA preserves full token-to-token interactions in real arithmetic, making it a true drop-in replacement for pretrained models such as CLIP and SAM with zero retraining.

All benchmarks are conducted on a single NVIDIA A100 (40 GB), averaged over 200 runs. ELSA's throughput advantage over ME-SDPA widens significantly with sequence length, reflecting the growing penalty of $O(n)$ sequential depth at scale.
